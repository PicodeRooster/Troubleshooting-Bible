## YAML Frontmatter — The Problem & The Plan

Learned today that Obsidian and Lunar.js both benefit from (or outright depend on) YAML frontmatter in `.md` files for things like metadata tagging, search indexing, and linking entries back to the master `index.html`. The issue: none of the existing articles were written with frontmatter in mind. They're clean prose — no `---` blocks, no `title:`, `tags:`, `category:`, or `date:` fields.

Manually going back through every existing file to add this by hand is not the move. That's error-prone, tedious, and doesn't scale.

---

## The Solution: A Frontmatter Injection Script

The plan is to write a **reusable bash (or Python) script** that can be run in two scenarios:

1. **Retroactively** — walks through all existing `.md` files and prepends a YAML block based on whatever can be inferred from the file (filename as title, folder as category, today's date, etc.)
2. **Pre-publish** — runs on any new article before it goes into the Bible, so authors can keep writing in plain markdown without worrying about frontmatter at all

### What the Script Needs to Do

- Detect if a file already has a frontmatter block (starts with `---`) — **skip it if so**, don't double-inject
- Derive `title:` from the filename (strip hyphens/underscores, title-case it)
- Set `date:` to today
- Accept optional `tags:` either passed as arguments or prompted interactively
- Prepend the block cleanly and leave the rest of the file untouched

### Stretch Goals

- A `--dry-run` flag to preview what would be injected without writing anything
- A `--tags` flag for passing comma-separated tags at the command line
- Batch mode: point it at a directory and process all `.md` files recursively

---

## Why This Matters for the Offline Build

The end goal is a single downloadable zip (or standalone file) that runs entirely in the browser. Lunar.js uses the frontmatter fields to build its search index — without consistent metadata across files, search results will be inconsistent or broken. Getting frontmatter right now, before the article count grows, keeps the publish pipeline clean going forward.
